# Общее расписание

Небольшой сайт для общего расписания Димы и Александры: личные дела, совместные предложения и фоновые события в одной временной шкале.

## Возможности

- текущее время и даты по Томску (`Asia/Tomsk`, UTC+7);
- счетчики общения, первой встречи и первого поцелуя;
- единое расписание без учебного расписания Димы;
- добавление, редактирование и удаление занятий с произвольной датой и временем;
- типы событий: личное/обычное, совместное предложение, фоновое событие;
- категории и фильтры: работа, личные дела, спорт, встреча, учеба, семья, фон, другое;
- фильтры по участникам: Дима, Александра, вместе, фон;
- фоновое расписание для гран-при Формулы, футбола и других событий, которые не должны мешать основным делам;
- горизонтальная шкала суток от `00:00` до `24:00`;
- SQLite-хранилище на сервере.

## Локальный запуск

```powershell
python server.py
```

После запуска откройте `http://localhost:8080`.

Для проверки произвольного момента по томскому времени можно использовать параметр `now`:

```text
http://localhost:8080/?now=2026-07-01T17:30:00
```

## Данные

События сохраняются в SQLite-файле `data/requests.sqlite3`. В Docker-контейнере файл вынесен в volume `/data`, чтобы данные не пропадали при обновлении сайта.

Основной API:

- `GET /api/events` - список событий;
- `POST /api/events` - создать событие;
- `PUT /api/events/{id}` - обновить событие;
- `DELETE /api/events/{id}` - удалить событие.

Старая ручка `/api/requests` оставлена только для совместимости с прошлой версией.
